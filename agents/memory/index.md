# Agent Memory Index

Agent memory is durable, repo-versioned context that prevents repeated
reasoning, repeated mistakes, or repeated file crawling.

Memory is not a diary, PR log, transcript, scratchpad, or status journal.

## Folders

- `decisions/`: durable product, process, and architecture decisions.
- `lessons/`: repeated blockers, rejected PR lessons, bugs, or CI lessons.
- `reviewer-patterns/`: recurring reviewer expectations.
- `security/`: security, privacy, prompt-injection, and secret-handling lessons.

## Memory File Shape

Each memory file should include:

- title,
- status: `active`, `superseded`, or `deprecated`,
- date created or updated,
- lanes and tags,
- summary,
- rule of thumb,
- source or context links,
- supersedes or superseded-by when relevant.
